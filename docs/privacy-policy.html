1. Introduction

Welcome to PixelQR ("we," "our," or "us"). We are committed to protecting your privacy and ensuring the security of your personal information. This Privacy Policy explains how we collect, use, and safeguard information when you use our browser extension.

PixelQR is a browser extension that allows you to generate, customize, and scan QR codes. We prioritize your privacy and have designed our extension to work offline for core features, minimizing data collection.

2. Information We Collect

2.1 Information You Provide

We collect the following information when you use PixelQR:

  • QR Code Content: The text, URLs, or other content you use to generate QR codes. This information is stored locally on your device and is never transmitted to our servers.
  • User Preferences: Your customization settings (colors, sizes, logos) are stored locally in your browser's storage.
  • History Data: Your QR code generation and scan history is stored locally on your device.

2.2 Google Drive Integration (Optional)

Important: Google Drive integration is an optional feature. We only collect data when you actively choose to use this feature.

When you use the Google Drive integration feature, we collect:

  • Google Access Token: Stored locally in your browser to authenticate with Google Drive API. This token is only used to access Google Drive services on your behalf.
  • User Profile Information: Basic information (name, profile picture) retrieved from Google for display purposes only. This information is cached locally and not transmitted to third parties.
  • File Metadata: Information about files you upload (filename, size, type) is used only to generate QR codes for sharing links.

2.3 Information We Do NOT Collect

We do NOT collect:

  • Your browsing history or web activity
  • Personal identification information (unless you choose to use Google Drive integration)
  • Content of files uploaded to Google Drive (we only access file metadata)
  • Your existing Google Drive files (we only access files created through this extension)
  • Any data transmitted to third-party servers (except Google APIs when using Drive integration)

3. How We Use Your Information

3.1 Core Features (Offline)

For core QR code generation and scanning features:

  • All processing happens locally in your browser
  • No data is transmitted to external servers
  • Your information remains on your device

3.2 Google Drive Integration

When you use Google Drive integration:

  • Authentication: We use Google OAuth2 to securely authenticate your Google account
  • File Upload: We upload files to your Google Drive and generate sharing links
  • QR Code Generation: We create QR codes for the generated sharing links
  • Display: We display your profile information (name, avatar) in the extension interface

All Google Drive operations use the following scopes:

  • https://www.googleapis.com/auth/drive.file - Only access files created through this extension
  • https://www.googleapis.com/auth/userinfo.profile - Only read basic profile information

4. Data Storage

All data is stored locally on your device using your browser's storage APIs:

  • chrome.storage.local (Chrome/Edge) or browser.storage.local (Firefox)
  • Data includes: QR code history, user preferences, Google access tokens (if using Drive integration)
  • No data is stored on external servers
  • You can clear all data by uninstalling the extension or clearing browser storage

5. Third-Party Services

5.1 Google Services

When you use Google Drive integration, we interact with the following Google services:

  • Google OAuth2 API: For user authentication
  • Google Drive API: For file upload and management
  • Google User Info API: For retrieving basic profile information

These services are governed by Google's Privacy Policy and Terms of Service. We recommend reviewing Google's privacy practices at https://policies.google.com/privacy.

5.2 Browser Extension Stores

When you install PixelQR from browser extension stores (Chrome Web Store, Firefox Add-ons, Edge Add-ons), your interaction with these stores is governed by their respective privacy policies.

6. Permissions

PixelQR requires the following permissions:

6.1 Required Permissions

  • activeTab: To access the current page URL for QR code generation
  • contextMenus: To provide right-click menu options
  • storage: To save your preferences and history locally
  • downloads: To download generated QR code images
  • scripting: To inject content scripts for page interaction
  • favicon: To retrieve website icons for display

6.2 Optional Permissions

Identity Permission (Optional):

The identity permission is only requested when you choose to use Google Drive integration. This permission is:

  • Not required for core QR code features
  • Only used for Google OAuth2 authentication
  • You can use the extension without granting this permission
  • You can revoke this permission at any time

7. Your Rights and Choices

7.1 Data Access and Control

You have the following rights:

  • Access: View all data stored by the extension in your browser's storage
  • Delete: Clear extension data by uninstalling the extension or using browser storage management
  • Control: Choose whether to use Google Drive integration

7.2 Google Account Control

If you use Google Drive integration, you can:

  • Revoke access through the extension's user interface
  • Revoke access through your Google Account settings at https://myaccount.google.com/permissions
  • Sign out from the extension at any time

8. Data Security

We implement the following security measures:

  • All data is stored locally on your device
  • Google OAuth2 tokens are encrypted and stored securely in browser storage
  • No sensitive data is transmitted to our servers (we don't operate any servers)
  • All API communications use HTTPS encryption
  • We follow Google's security best practices for OAuth2 implementation

9. Children's Privacy

PixelQR is not intended for children under the age of 13. We do not knowingly collect personal information from children. If you believe we have collected information from a child, please contact us immediately.

10. Changes to This Privacy Policy

We may update this Privacy Policy from time to time. We will notify you of any changes by:

  • Updating the "Last Updated" date at the top of this page
  • Posting a notice in the extension (for significant changes)
  • Updating the extension's store listing

Your continued use of PixelQR after any changes constitutes acceptance of the updated Privacy Policy.
